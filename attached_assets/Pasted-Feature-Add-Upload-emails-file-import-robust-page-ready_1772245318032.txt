Feature: Add "Upload emails" file-import + robust page-ready wait

Goal
- Add a button that lets users upload a file containing a list of emails (same UX flow as the existing "Import links" feature).
- Parse file contents into emails, validate and deduplicate, then pass the email list to the same import handler used for links (or to a new `importEmails(emails)` function if preferred).
- Make the client script wait until the page/form is fully loaded (and optionally until a given DOM element exists) before running the import logic to avoid "form missing" or timing errors. Provide configuration for a 3s or 5s extra wait.

Requirements / Acceptance criteria
- UI:
  - Button: "Upload emails" next to "Import links".
  - Accepts `.txt`, `.csv`. (One email per line OR comma/semicolon separated).
  - Shows progress / success / invalid emails list / count of deduped items.
- Behavior:
  - Parse file, trim entries, validate with email regex, remove duplicates and empty lines.
  - If any invalid emails, display them and allow user to continue (import only valid) or cancel.
  - Use the same backend/import pipeline as links (call `importLinks(...)` or `importEmails(...)`).
- Robustness:
  - Ensure script waits until DOM is `ready` and the form element exists before attaching event handlers. Default to an extra wait of 3000ms (configurable to 5000ms).
  - Fallback: if element doesn’t appear within 10s, show a clear error message in UI.
- Tests:
  - Upload sample `.txt` and `.csv`.
  - Upload a file with invalid emails and verify the UI shows the invalid list.
- Performance/security:
  - Limit file size (e.g. 2MB client-side) and validate server-side again.
  - Sanitize inputs before sending to backend.

Deliverables
- UI code (HTML/CSS/JS) for the button and file input.
- Integration code calling existing import handler.
- Unit / manual test plan and example files.
